# 数据mock

在整体项目中，所有请求的mock都通过http.js文件。

  1. 目录介绍
    ├── mock                       
│   ├── api                        # 所有请求
│   │   ├── content                # 内容管理api
│   │       ├── index.js           # 内容管理具体的api请求
│   ├── data                       # mock的所有数据
│   │   ├── content                # 内容管理data
│   │       ├── articleList.js     # 文章列表数据
│   │       ├── cloumn.js          # 文章栏目数据
│   ├── index.js                   # 入口文件
│   ├── mock-server.js             # 配置文件
│   ├── remote-search.js    
├── src                       
│   ├── utils                      # 通用 
│   │   ├── http.js                # http.js请求文件
    • mock/api文件夹：
      所有的请求放到该文件夹下。具体规则：如“内容管理”，需要在api文件夹下新建content文件夹，在content文件夹下新建index.js文件，index.js里面写api。
    • mock/data文件夹：
      所有mock的数据放到data文件夹下。具体规则：如“内容管理”，需要在data文件夹新建content文件夹，在content文件夹新建各个需要mock的数据的文件，例如文章列表数据articleList.js，文章栏目数据cloumn.js。
    • mock/index.js文件： 主文件，所有的api都需要引入到这个文件夹下。如刚刚建立的api/content/index.js需要引入到该文件中：import content from './api/content'。
    • src/utils/http.js文件： 这个文件里面提供mock的请求postDev。

  1. 代码实现

    下面实现一个具体的数据mock（以文章列表为例）:

    • 建立mock的数据：在mock/data文件夹下新建content/articleList.js，将mock的数据写入：
    export default {
  'content': [{
    id: '1',
    contentTitle: '文章标题1',
    categoryId: '1',
    categoryName: '介绍',
    author: 'qhy',
    updateTime: 1564632316,
    state: 1,
    contentText: '富文本编辑器里面的内容修改1'
  }, {
    id: '2',
    contentTitle: '文章标题2',
    categoryId: '11',
    categoryName: '自然',
    author: 'qhy',
    updateTime: 1564632316,
    state: 0,
    contentText: '富文本编辑器里面的内容修改2'
  }],
  'last': true,
  'totalPages': 1,
  'totalElements': 2,
  'first': true,
  'sort': null,
  'numberOfElements': 2,
  'size': 10,
  'number': 0
}
    • 建立api：在mock/api文件夹下新建content/index.js，将api写入：
    // 内容管理mock的数据要在此处引入
import articleList from '../../data/content/articleList.js'
export default [
  {
    url: '/mock/content/getArticleList',
    type: 'post',
    response: _ => {
      return {
        code: 0,
        message: 'success',
        value: articleList
      }
    }
  }
]

    方法介绍

    mock接口的命名规则：使用"/mock/文件夹名/具体接口名"的命名规范

    • 将api引入index.js文件中：
    import content from './api/content'
const mocks = [
  ...content
]
    • 将接口写入src/api/protocol.js文件中（目的是方便切换mock的接口与真实接口）：
    param_getArticleList: {
  url: '/mock/content/getArticleList',
  data: {
    page: 1, // 第几页
    size: 15, // 每页搜索结果个数
    sort: [{
      direction: 'ASC',
      property: 'lowerNum'
    }],
    columnId: null, // 所属栏目id 可不传 1是文章 2轮播图
    contentTitle: null // 文章标题 模糊查询
  }
}
    • 调用mock的接口，例如在src/views/manage-page/content-manage/article-manage/components/modelRight.vue中请求：
    http.postDev(protocol.param_getArticleList).then(res => {
  const { code, message, value } = res.data
  console.log(code)
  console.log(message)
  console.log(value)
})

    方法介绍

    http.postDev请求的是mock的接口，http.post请求的是真实的接口。
    联调真实接口时只需要将http.postDev改为http.post，然后将src/api/protocol.js文件中的接口名改为真实的接口名即可。

    WARNING

    如果增加了代理，联调真实接口时需要注意，真实的接口不可以和mock的接口重名，如果重名会请求到mock的接口数据，真实的接口数据将请求不到。

    • postDev介绍，postDev方法在src/utils/http.js文件中：
    postDev({
  url,
  data
}) {
  const sessionid = store.state.user.UserUserId
  const sessionstr = store.state.user.UserSessionstr
  const Authorization = sessionid + ',' + sessionstr + ',' + url

  if (sessionid != null && sessionid !== '') {
    data.userStr = sessionid
    data.sessionStr = sessionstr
    return axios({
      method: 'post',
      baseURL: window.g.APP_URL_PATH_DEV, // 将baseUrl替换成本地的地址
      url: url,
      data: data,
      dataType: 'json',
      headers: {
        Authorization,
        'Session-Str': sessionstr,
        'User-Id': sessionid,
        'Accept-Language': Cookies.get('language') || 'en'
      },
      timeout: 120000
    })
      .then(response => {
        return checkStatus(response)
      })
      .then(res => {
        return checkCode(res)
      })
  } else {
    return axios({
      method: 'post',
      baseURL: window.g.APP_URL_PATH_DEV, // 将baseUrl替换成本地的地址
      url: url,
      data: data,
      dataType: 'json',
      timeout: 120000
    })
      .then(response => {
        return response
      })
      .then(res => {
        return checkCode(res)
      })
  }
}

    方法介绍

    postDev方法与post方法的不同之处在于将baseURL替换成本地的地址了。
    window.g.APP_URL_PATH_DEV对应的是本地地址栏的地址，对应到这个框架就是APP_URL_PATH_DEV: 'http://localhost:9999/dev-api'。
