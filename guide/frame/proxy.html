# 跨域问题

  1. 产生原因

    因为浏览器的同源策略规定某域下的客户端在没明确授权的情况下，不能读写另一个域的资源。所以在开发过程中经常会遇到跨域问题，跨域问题的报错一般如下图所示：

  2. 解决方法

    解决跨域问题需要本地联调时增加代理。增加代理的过程如下：

    • 找到根目录下的vue.config.js文件，在该文件下找到proxy，在该对象下新增如下代码：
    proxy: {
  [process.env.VUE_APP_BASE_API]: {
    target: `http://127.0.0.1:${port}/mock`,
    changeOrigin: true,
    pathRewrite: {
      ['^' + process.env.VUE_APP_BASE_API]: ''
    }
  },
  // 新增的代码
  // 在请求时铺捉到console将console以及console前面的地址替换成http://192.168.1.13/console
  '/console': {
    target: `http://192.168.1.13/console`, // 要访问的服务器域名
    changeOrigin: true, // 改变源
    pathRewrite: {
      '^/console': '' // console替换为空
    }
  }
},
    • 修改public文件夹下的config.js文件：
    window.g = {
  APP_URL_PATH: 'http://localhost:1221', // 将要访问的服务器域名改成本地的域名
  ···
  ···
  ···
}
    • 进入项目终端，重新npm run dev，再次请求就不会有报错了。
