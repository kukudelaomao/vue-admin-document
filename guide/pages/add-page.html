# 添加页面

在实际的开发过程中，经常需要新增页面，下面来介绍一下新增页面的过程。
拿初始框架举例，如果想在下图信息管理后面新增一个站内信管理：

  1. 新增页面模板

新建站内信文件模板（这里直接拷贝文章管理的模板，具体页面模板的介绍见页面模板章节）。

在src/views/manage-page下面新建文件夹station-mail，再在station-mail文件夹下新建station-mail-list
将src/views/manage-page/content-manage/article-manage文件夹下的index和components拷贝到src/views/manage-page/station-mail/station-mail-list文件夹下，如下三张图所示：

修改station-mail/station-mail-list/components/modelRight.vue，如下图所示：

<template>
  <div>站内信管理</div>
</template>
<script>
export default {

}
</script>
  1. 在本地路由表中新增该页面路由

在src/router/index.js中新增：

{
  rightCode: 'station-mail', // 权限编码
  component: Layout, // 一级路由 需要将component设置成Layout
  alwaysShow: true,
  name: '站内信管理' // 导航标题
},
{
  rightCode: 'station-mail-list', // 权限编码
  component: () => import('@/views/manage-page/station-mail/station-mail-list'), // 页面的位置
  alwaysShow: true,
  name: '站内信列表' // 导航标题
}
  1. 新增页面权限

新增权限分为两种情况，一种是在还没有联调权限接口也就是还没有真实的页面权限时，另一种是已经联调完权限接口时。

  • 还没有真实的权限时，新增权限需要修改mock数据：

在mock/data/user/permission.js中新增：

{
  'path': 'station-mail',  // 路径
  'component': '站内信管理',
  'children': [{
    'path': 'station-mail-list', // 子路径
    'component': '站内信列表',
    'children': [],
    'sortId': 22,
    'meta': {
      'rightIcon': null,
      'rightCode': 'station-mail-list',
      'title': '站内信列表'
    },
    'name': 'station-mail-list',
    'id': 14
  }],
  'sortId': 5,
  'meta': {
    'rightIcon': 'email',
    'rightCode': 'station-mail', // rightCode与path相同
    'title': '站内信管理'
  },
  'name': 'station-mail',
  'id': 4
}

修改了mock的数据需要重新启动项目，重新启动项目后就能看到站内信管理页面已经添加上了。

  • 联调真实接口后，通过接口将权限加上去。

权限编码对应的是src/router/index.js文件中的rightCode。

权限添加后，将权限分配给角色，被分配的角色就可以看到该页面了。
