# 分页

分页是针对表格数据来进行的分页。目前分页有两种情况：从1开始的分页和从0开始的分页。

  • 从1开始的分页的项目主要有：谷得、大象、应价、积分商城等。
  • 从0开始的分页的项目主要有：投顾。
  1. 页面代码及解析
// 通过import引入分页组件（新框架中会写成全局组件，不必每个页面都引用）
import pagination from '@/components/until/pagination'

<!-- 分页组件 -->
<!-- :totals="total" 数据总量 -->
<!-- :page="page" 当前页码 -->
<!-- :init-page="initPage" 初始页码 -->
<!-- :page-size="pageSize" 每页展示的数据量 -->
<!-- @gotoPage="gotoPage" 跳转页面时触发的方法 -->
<!-- @changePageSize="changePageSize" 改变每页显示的数据条数时触发的方法 -->
<pagination 
  :totals="total" 
  :page="page" 
  :init-page="initPage" 
  :page-size="pageSize" 
  @gotoPage="gotoPage" 
  @changePageSize="changePageSize" 
/>

在data()和method()里面定义数据和方法：
定义数据：

data() {
  return {
    total: 1,
    initPage: 0,
    page: 0,
    pageSize: this.protocol.param_findContent.data.size
      ? this.protocol.param_findContent.data.size : 15,
  }
}

initPage和page

initPage和page相等。如果服务端要求页码从0开始，initPage和page都为0，如果服务端要求页码从1开始，initPage和page都为1。

pageSize赋值解析

this.protocol.param_findContent.data.size ? this.protocol.param_findContent.data.size : 15
如果protocol.param_findContent.data.size里面有值，就用protocol.param_findContent.data.size里面的值，否则就显示15条。
这样写是因为如果用户改变了每页显示的数据数量，protocol.param_findContent.data.size可以记录下来，在用户再次点击搜索的时候，每页的数据数量按照用户改变后的数量显示。

定义方法：

methods: {
  gotoPage(page) {
    // 如果initPage为0，那么将page减去1
    const gotoPage = this.initPage === 0 ? page - 1 : page
    // 跳转页面
    this.postFn(gotoPage)
  },
  changePageSize(size) {
    // 改变每页显示的数据条数后，将size赋值给接口参数
    this.protocol.param_findContent.data.size = size
    // 重新跳转到第一页
    this.postFn(this.initPage)
  }
}
  1. 分页组件代码及解析
    分页组件位于src/components/until/pagination.vue文件：
<template>
  <el-footer style="height:30px;margin-top:10px;">
    <el-pagination
      :current-page="initPage === 0 ? page + 1 : page"
      :page-sizes="[15, 20, 50, 100]"
      :page-size="pageSize"
      layout="total, pager, jumper, sizes"
      background
      :total="totals"
      @size-change="handleSizeChange"
      @current-change="handleCurrentChange"
    />
  </el-footer>
</template>
<script>
export default {
  name: 'Pagination',
  props: {
    'totals': {
      type: Number,
      default: 0
    },
    'initPage': {
      type: Number,
      default: 0
    },
    'page': {
      type: Number,
      default: 0
    },
    'pageSize': {
      type: Number,
      default: 15
    }
  },
  methods: {
    handleSizeChange(val) {
      // 调用父组件的changePageSize方法
      this.$emit('changePageSize', val)
      console.log(`每页 ${val} 条`)
    },
    handleCurrentChange(val) {
      // 调用父组件的gotoPage方法
      this.$emit('gotoPage', val)
      console.log(`当前页: ${val}`)
    }
  }
}
</script>

方法解析

分页组件接收父组件传过来的totals，initPage，page，pageSize参数。
在用户改变页码时，调用父组件的gotoPage方法。
在用户改变每页显示的条数时，调用父组件的changePageSize方法。
